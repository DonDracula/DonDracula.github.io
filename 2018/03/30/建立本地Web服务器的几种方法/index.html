建立本地Web服务器的几种方法

2018.03.30 Liu Yanhao Web开发  热度 ℃

查看web生成项目的示例，一般有两种方法。
1.可以直接在浏览器中打开解压或克隆目录的html文件，运行某个示例；
2.可以本地安装一个Web服务器。对于大多数的基础性示例，方法1就可以了，但当我们需要加载一些外部资源时，例如三维模型和纹理图片等，仅打开一个HTML文件是不够的。
这种情况下我们就需要一个本地Web服务器，确保外部资源能正确加载。
下面列出了几个简单建立Web服务器的例子。

1. 用Python————适合大多数UNIX/Mac系统

大多数的UNIX/Linux/Mac系统都已经安装了Python。在这些系统中可以很容易建立本地Web服务器：

> python -m SimpleHTTPServer
Serving HTTP on 0.0.0.0 port 8000 …

在资源示例源代码目录下运行上述命令。

2. 用NPM——如果已经安装了Node.js

如果已经用过Node.js，那么很可能系统中已经安装了NPM。使用NPM时有两种选择，他们都可以快速建立用于测试的本地Web服务器。
第一种是使用HTTPServer模块：

> npm install -g http-server
> http-server
Starting up http-server, serving ./ on port: 8080
Hit CTRL-C to stop the server

也可以用SimpleHTTPServer

> npm install -g simple-http-server
> nserver
Simple-http-server Now Serving： /Users/jos/git/Physijs at http://
Localhost：8000/

但是第二种方式有个缺点，他不能自动的列出文件夹里东西，而第一种方式就可以。

3. 用Mongoose———在Mac/Windows上可以移植

如果没有安装Python或者NPM，那么还可以用一个简单的，可移植的Web服务器，Mongoose。
首先，从https://code.google.com/p/mongoose/downloads/list 下载特定平台的二进制文件。
如果是Windows平台，把下载好的文件复制到示例所在的目录，双击即可启动一个Web浏览器，并显示该目录中的内容。
对于其他操作系统，也必须将可执行程序复制到目标目录，通过命令行来启动。这两种情况下，本地Web服务器都会启动，端口是8080。

点击相应的文件，就可以直接测试了。
